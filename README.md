# Telegram Bot — MiniApp + Game (Coin Collector)

ربات تلگرام پیشرفته با یک مینی‌اپ بازی و امکانات مدیریتی و فروشگاه داخلی.

این شاخه (feature/miniapp-game) شامل افزونه‌ها و بهبودهای زیر است:

- بازی "Coin Collector" — سه صندوق با احتمال‌ها و جوایز مختلف
- سیستم ذخیره‌سازی محلی با SQLite (کاربران، تاریخچه بازی، موجودی آیتم‌ها)
- فروشگاه درون‌رباتی با آیتم‌هایی مانند کلید شانس و شیلد
- آیتم‌ها تاثیر مستقیم روی بازی: کلید شانس احتمال جایزه بزرگ را بالا می‌برد، شیلد از جریمه جلوگیری می‌کند
- جوایز روزانه، لیدربورد، پروفایل کاربر و تاریخچه‌ی بازی
- محدودیت زمانی (cooldown) برای جلوگیری از بازی پشت‌سرهم
- دستورات مدیریتی (ADMINS): /setbalance, /addcoins, /export, /reset
- Dockerfile و docker-compose برای استقرار ساده
- تست‌های پایه و GitHub Actions CI برای اجرای تست‌ها

---

ویژگی‌ها (فهرست مفصل)

- تجربهٔ کاربری فارسی با کیبوردهای اینلاین و پیام‌های واضح
- جریان کامل بازی: /menu → 🎮 مینی‌اپ → انتخاب صندوق → مشاهده نتیجه
- فروشگاه: /shop یا منوی اصلی → خرید آیتم، ذخیره در اینونتوری و نمایش در /profile
- امنیت ساده: ADMINS در config برای دستورات مدیریتی
- مهاجرت سادهٔ دیتابیس: ensure_db در شروع برنامه ساخت/به‌روزرسانی جداول را انجام می‌دهد
- خروجی CSV برای داده‌ها (برای بکاپ یا تحلیل) از طریق /export (ادمین)
- Docker-friendly: می‌توانید کانتینر بسازید و اجرا کنید یا از docker-compose استفاده کنید

---

دستورهای مهم ربات

- /start — شروع و ثبت کاربر
- /menu — باز کردن منوی اصلی
- /miniapp — ورود به بازی
- /shop — فروشگاه و خرید آیتم
- /profile — نمایش پروفایل و آیتم‌ها
- /balance — مشاهده موجودی
- /leaderboard — مشاهده لیدربورد
- /daily — دریافت جایزهٔ روزانه (هر 24 ساعت)
- /setbalance <user_id> <amount> — تنظیم موجودی (فقط ADMINS)
- /addcoins <user_id> <amount> — اضافه کردن سکه (ADMINS)
- /export — خروجی CSV از داده‌ها (ADMINS)
- /reset و /reset_confirm — بازنشانی دیتابیس (ADMINS)

---

پیکربندی و فایل‌های مهم

- config_example.py — الگوی تنظیمات. از آن یک فایل config.py بسازید و BOT_TOKEN و ADMINS را تنظیم کنید.
- bot.py — ورودی و هندلرهای اصلی
- storage.py — مدیریت SQLite، خرید/مصرف آیتم، لاگ بازی
- game.py — منطق بازی و اثر آیتم‌ها
- ui.py — کیبوردهای اینلاین
- Dockerfile و docker-compose.yml — برای deploy

---

نحوه اجرا (محلی)

1. Python 3.10+ نصب کن
2. یک فایل `config.py` بساز (بر اساس `config_example.py`) و مقدار `BOT_TOKEN` و در صورت نیاز `ADMINS` را تنظیم کن
3. وابستگی‌ها را نصب کن:

   pip install -r requirements.txt

4. اجرای بات در حالت محلی:

   python bot.py

5. یا با داکر:

   docker build -t telegram-bot .
   docker run -e BOT_TOKEN=YOUR_TOKEN -it telegram-bot

یا با docker-compose (مطابق `docker-compose.yml` و فایل `.env`):

   docker compose up --build -d

---

تست و CI

- تست‌های اولیه در پوشه `tests/` قرار دارند. برای اجرای تست‌ها:

  pytest -q

- GitHub Actions اولیه برای اجرای تست‌ها اضافه شده است (`.github/workflows/ci.yml`).

نکات امنیتی و نگهداری

- BOT_TOKEN را همیشه به صورت متغیر محیطی نگه دارید؛ آن را در ریپوزیتوری عمومی قرار ندهید.
- فایل دیتابیس (`bot.db`) را در `.gitignore` قرار دارد تا به مخزن ارسال نشود.
- برای مقیاس‌پذیری و امنیت بهتر، در محیط تولید از یک دیتابیس ریموت (Postgres/MySQL) و مکانیزم لاگ‌گذاری مناسب استفاده کنید.
